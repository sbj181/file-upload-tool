## Concerns

1. **`.env.example` key mismatch vs. plan literal text.** The plan's Task 0.2 Step 3 code block
   uses `EMAIL_FROM=__VERIFIED_RESEND_SENDER__`, but the task brief and the real `.env.local`
   both use `FROM_EMAIL`. I followed the brief/reality (`FROM_EMAIL`) rather than the plan's
   literal typo, and confirmed no code in these three tasks references either name. Flagging so
   later phases (email sending, Task 2.3) use `FROM_EMAIL` consistently.
2. **`.env.example` omits legacy Google OAuth vars.** The real `.env.local` still carries
   `ADMIN_EMAIL`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REDIRECT_URI`, `GOOGLE_REFRESH_TOKEN` from the
   old googleapis-based flow. These aren't part of Task 0.2's listed keys and the new
   `google-auth-library`-based ID-token verification (Phase 4) doesn't need them, so they were
   left out of `.env.example`. If any later phase still needs them, `.env.example` should be
   updated then.
3. **Pre-existing typecheck errors** are expected and unrelated to this work (see above) ‚Äî
   confirmed none touch the three new lib files.
4. **`pnpm` lockfile warning during `remove`**: pnpm printed `WARN Ignoring broken lockfile ...
   not compatible with current pnpm` on the first `pnpm remove` call. It self-healed by
   regenerating the lockfile; `pnpm test` and subsequent installs worked fine afterward. Worth a
   glance if lockfile drift causes CI friction later.
5. Working directory `.superpowers/` is untracked in git (present but not part of any commit in
   this session) ‚Äî left as-is since it's outside the scope of these tasks.
